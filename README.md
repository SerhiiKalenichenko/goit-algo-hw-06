# 1. Модель графа

Створено неорієнтований зважений граф, що моделює транспортну мережу району міста:
Вершини: A–H (перехрестя).
Ребра: дороги між вершинами.
Ваги ребер: умовна довжина маршруту (км).
Граф зібрано у файлі graph_model.py і використовується у всіх завданнях.

## 2. Завдання 1

Побудова та аналіз графа

Файл: task_1.py

Реалізовано:
створення графа через networkx.Graph;
візуалізація графа з підписами ваг ребер;
аналіз характеристик:
кількість вершин та ребер;
ступені вершин;
значення ваг ребер.

### 3. Завдання 2

DFS та BFS

Файл: task_2.py

Виконано:
реалізація DFS (глибина) та BFS (ширина);
пошук шляхів між вершинами;
порівняння результатів алгоритмів.

Короткі висновки:
DFS рухається глибше першою доступною гілкою → може давати довший шлях.
BFS проходить рівнями → завжди знаходить шлях із мінімальною кількістю ребер.

#### 4. Завдання 3

Алгоритм Дейкстри

Файл: task_3.py

Реалізовано:
власна імплементація алгоритму Дейкстри через heapq;
пошук найкоротших шляхів від кожної вершини до всіх інших;
відновлення шляху через батьківські вершини;
вивід повної таблиці найкоротших маршрутів.
